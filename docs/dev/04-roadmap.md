# 路线图

> 状态: 已完成 v0.1.0，后续版本为计划交付内容。

## 版本规划

### v0.1.0 - 协议基础 ✅

已完成: 项目初始化、MQTT v3.1.1 与 v5.0 协议编解码 (含属性系统)

---

### v0.2.0 - 单节点 Broker 🚧

**预计**: 3-4 周

核心: TCP/WebSocket 服务器、会话管理、主题管理、消息路由 (QoS 0/1/2)、SeaORM + PostgreSQL 持久化

质量目标: 10,000+ 并发连接，P99 < 50ms，测试覆盖率 > 80%

---

### v0.3.0 - 安全与认证

**预计**: 2-3 周

核心: TLS 1.2/1.3、证书认证、用户名/密码/Token 认证、ACL 授权控制

---

### v0.4.0 - API 接口

**预计**: 2 周

核心: REST API (Axum) — 客户端/主题/系统管理，Prometheus 指标，健康检查

---

### v0.5.0 - 规则引擎

**预计**: 3 周

核心: 规则解析器、消息/内容路由、数据转换、条件触发、钩子插件、HTTP Webhook

---

### v0.6.0 - Web GUI

**预计**: 3-4 周

技术栈: Vue 3 + Reka UI + Pinia + Vite

核心: 仪表盘、客户端管理、主题管理、规则配置、系统设置、WebSocket 实时推送

---

### v0.7.0 - Raft 集群

**预计**: 4-5 周

核心: Raft 共识 (Leader 选举、日志复制)、gRPC 节点通信、状态同步、故障转移

质量目标: 3/5/7 节点集群，故障恢复 < 30s

---

### v1.0.0 - 生产就绪

**预计**: 2-3 周

核心: 集成/压力/稳定性测试、Docker/Kubernetes/Helm 部署、完整文档

质量目标: 100,000+ 并发连接，100,000+ msg/s，99.99% 可用性

## 后续规划

### v1.x

- 外部集成: Kafka、RabbitMQ
- 边缘计算: 边缘节点支持
- 多租户: 租户隔离和资源配额
- 高级规则引擎: 复杂事件处理 (CEP)

### v2.x

- MQTT 5.0 完整特性: 共享订阅、流控
- 全球分布式: 多区域部署
- 协议插件: CoAP、LwM2M
