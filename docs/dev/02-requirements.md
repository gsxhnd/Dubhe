# 需求分析

> 状态: 规划中。描述目标产品需求，不代表已实现。

## 用户角色

| 角色 | 描述 |
|------|------|
| IoT 设备 | 通过 MQTT 协议连接的物联网设备 |
| 应用开发者 | 使用 MQTT 构建应用的开发人员 |
| 系统管理员 | 负责部署、配置和运维的人员 |
| 运维工程师 | 负责监控和故障处理的人员 |

## 功能需求

### 协议支持

- MQTT v3.1.1: 完整实现所有控制包类型
- MQTT v5.0: 完整实现，包括属性系统、共享订阅
- MQTT over WebSocket / WSS

### Broker 核心

- 客户端连接管理 (TCP/WebSocket)
- 会话管理 (Clean Session / Persistent Session)
- 主题管理 (主题树、通配符 +/#)
- 消息路由 (QoS 0/1/2 完整流程)
- 保留消息、遗嘱消息、Keep Alive

### 安全

- TLS 1.2/1.3 加密传输
- 用户名/密码、客户端证书、Token 认证
- 基于 ACL 的主题访问控制
- 发布/订阅权限分离
- 匿名连接控制

### 客户端管理

- 在线/离线客户端列表与详情
- 流量统计、连接历史
- 强制断开、清除会话
- 分组、标签、黑白名单

### 消息持久化

- QoS 1/2 消息持久化 (PostgreSQL)
- 会话、订阅、保留消息持久化
- 消息历史查询

### 规则引擎

- 消息路由规则 (基于主题、内容条件)
- 数据转换 (JSON 字段提取、格式转换)
- 条件触发 (阈值告警、事件触发)
- HTTP Webhook 转发
- 预留外部集成接口 (Kafka、RabbitMQ)

### 插件系统

- 认证插件: 内置密码认证、JWT、自定义接口
- 授权插件: ACL 规则、文件/数据库授权、自定义接口
- 钩子插件: on_connect、on_disconnect、on_publish、on_subscribe、on_unsubscribe

### API 接口

- REST API (Axum): 客户端/主题/订阅/规则/系统/监控
- Prometheus 指标导出

### Web GUI

- 系统仪表盘、客户端管理、主题浏览器
- 规则引擎配置、集群状态监控、系统设置
- WebSocket 实时数据推送

### 集群支持

- Raft 共识协议 (Leader 选举、日志复制)
- gRPC 节点间通信 (Tonic)
- 状态同步、故障自动转移
- 支持 3/5/7 节点部署

## 非功能需求

### 性能

- 单节点 100,000+ 并发连接
- 消息吞吐量 100,000+ msg/s
- 消息延迟 P99 < 10ms
- 每连接内存占用 < 10KB

### 可靠性

- 单节点 99.9%，集群 99.99% 可用性
- 集群故障恢复 < 30s

### 可扩展性

- 水平扩展 (增加集群节点)
- 插件化架构
- 模块化代码结构

### 可运维性

- 结构化日志 (tracing)
- Prometheus 指标导出
- 健康检查接口
- 配置热更新
- Docker/Kubernetes 支持

### 安全性

- TLS 加密、认证授权
- 防暴力破解、连接速率限制

## 暂不支持

- Kafka/RabbitMQ 集成 (后续版本)
- 协议插件 (CoAP、LwM2M)
- 多租户
- 全球多区域部署
