# UI 设计

> 状态: 设计草案。当前仓库未包含前端实现。

## 设计目标

- 简洁直观: 清晰的信息层级
- 实时响应: WebSocket 数据推送
- 响应式: 适配桌面和平板
- 暗色主题: 支持明暗切换

## 技术栈

| 组件 | 技术 |
|------|------|
| 框架 | Vue 3 |
| UI 库 | Reka UI |
| 图表 | ECharts |
| 状态管理 | Pinia |
| 构建工具 | Vite |

## 页面结构

- 仪表盘 (Dashboard)
- 客户端 (Clients) — 在线客户端、会话管理
- 主题 (Topics) — 主题列表、保留消息
- 订阅 (Subscriptions)
- 集群 (Cluster) — 节点状态、Raft 信息
- 监控 (Monitoring) — 实时指标、告警配置
- 设置 (Settings) — 系统配置、安全设置、日志管理

## 仪表盘

关键指标卡片:

| 指标 | 描述 | 更新频率 |
|------|------|---------|
| 在线客户端 | 当前连接数 | 实时 |
| 主题数量 | 活跃主题总数 | 5s |
| 消息/秒 | 当前吞吐率 | 1s |
| 订阅数 | 总订阅关系 | 5s |

图表: 消息吞吐趋势 (折线)、连接数趋势 (面积)、热门主题 Top 10、系统资源使用率

## 客户端管理

列表页:
- 分页展示，按协议版本/连接状态筛选
- 搜索客户端 ID，批量断开

详情页:
- 基本信息: ID、IP、协议版本
- 会话信息: Clean Session、过期时间
- 订阅列表: 已订阅主题及 QoS
- 操作: 断开连接、清除会话

## 主题管理

- 树形结构展示主题层级
- 显示订阅者数量和消息频率
- 保留消息查看与删除

## 集群管理

| 字段 | 描述 |
|------|------|
| 节点 ID | 唯一标识 |
| 地址 | 网络地址 |
| 角色 | Leader / Follower |
| 状态 | 在线 / 离线 |
| 日志索引 | 复制进度 |

## 交互设计

- WebSocket 实时数据推送
- 通知: 连接/断开/集群变化/告警
- 快捷键: `Ctrl+K` 搜索, `Ctrl+R` 刷新, `Esc` 关闭弹窗

## 响应式断点

| 断点 | 宽度 | 设备 |
|------|------|------|
| md | >= 768px | 平板 |
| lg | >= 992px | 小桌面 |
| xl | >= 1200px | 桌面 |
| xxl | >= 1600px | 大桌面 |

布局: 桌面侧边栏 240px + 内容区自适应，小屏收起为抽屉式菜单。

## 可访问性

- 所有交互元素提供 ARIA 标签
- 支持完整键盘导航
- 焦点状态清晰可见

## 国际化

支持: 简体中文 (zh-CN)、English (en-US)
